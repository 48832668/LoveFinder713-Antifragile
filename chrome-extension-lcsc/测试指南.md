# 扩展状态检测功能测试指南

## 问题诊断

如果您看到 "内容脚本未加载或当前页面未匹配" 的提示，这表明content script没有在当前页面运行。以下是可能的原因和解决方法：

## 测试步骤

### 1. 基本测试
1. 打开一个LCSC商品详情页（例如：`https://item.szlcsc.com/xxx.html`）
2. 点击扩展图标打开popup
3. 点击"检测页面扩展"按钮
4. 观察状态栏的变化

### 2. 预期结果
- **成功**：状态显示"内容脚本在当前页面运行"（绿色）
- **部分成功**：状态显示"内容脚本运行但无匹配配置"（绿色，表示脚本运行但配置不匹配）
- **失败**：状态显示"内容脚本未加载或当前页面未匹配"（红色）

## 可能的问题和解决方案

### 问题1：页面URL不匹配
**症状**：显示"内容脚本未加载或当前页面未匹配"
**原因**：当前页面URL不符合manifest.json中配置的模式 `*://item.szlcsc.com/*.html*`
**解决**：
- 确保访问的是LCSC商品详情页（以`.html`结尾）
- 检查URL是否包含`item.szlcsc.com`

### 问题2：Content Script未注入
**症状**：显示"内容脚本未加载或当前页面未匹配"
**原因**：Chrome没有注入content script
**解决**：
1. 重新加载扩展
2. 刷新页面
3. 检查控制台是否有错误信息

### 问题3：权限问题
**症状**：显示"内容脚本未加载或当前页面未匹配"
**原因**：扩展权限配置有问题
**解决**：
1. 确保扩展已正确加载
2. 检查manifest.json中的权限配置
3. 重新安装扩展

### 问题4：Manifest V3兼容性
**症状**：测试无响应
**原因**：MV3的消息传递机制与MV2不同
**解决**：
1. 确保使用的是最新Chrome版本
2. 检查service worker配置

## 调试信息

### 查看控制台日志
1. 打开popup页面
2. 右键点击popup选择"检查"
3. 查看Console面板的日志信息

### Content Script日志
1. 在LCSC商品页面右键选择"检查"
2. 查看Console面板的日志
3. 应该看到"[LCSC读取器]"开头的日志信息

## 常见错误代码

- `Could not establish connection`：Content script未加载
- `Cannot access chrome.tabs`：权限问题
- `Invalid tab ID`：标签页状态问题

## 解决方案清单

1. **重新加载扩展**
   - 访问 `chrome://extensions/`
   - 找到扩展并点击"重新加载"

2. **清除缓存**
   - 在LCSC页面按Ctrl+Shift+R（Windows）或Cmd+Shift+R（Mac）

3. **检查扩展状态**
   - 确保扩展图标可见
   - 检查是否有错误提示

4. **重新安装扩展**
   - 在`chrome://extensions/`中卸载
   - 重新加载扩展文件夹
