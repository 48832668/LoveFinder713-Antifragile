# LCSC 订单解析 Chrome 扩展技术文档

## 一、整体架构概览

本扩展采用典型的 Chrome 扩展 MV3 架构，包含以下核心文件：

```
chrome-extension-lcsc-order/
├── manifest.json          # 扩展配置文件（入口）
├── scripts/
│   ├── content-script.js # 内容脚本（核心业务逻辑）
│   └── background.js     # 后台脚本（API 转发）
├── styles/
│   └── content-style.css # 样式文件
└── icons/                # 图标资源
```

## 二、各文件职责详解

### 2.1 manifest.json - 扩展配置入口

这是 Chrome 扩展的"身份证"，告诉浏览器扩展的基本信息和权限。

```json
{
  "manifest_version": 3,  // 使用 Chrome 扩展最新的 MV3 版本
  "name": "LCSC订单信息解析器",
  "version": "1.0.0",
  
  // 扩展权限
  "permissions": [
    "storage",      // 存储权限
    "activeTab"     // 活动标签页权限
  ],
  
  // 主机权限（允许访问的网站）
  "host_permissions": [
    "https://member.szlcsc.com/*",   // LCSC 会员订单页面
    "https://www.szlcsc.com/*",      // LCSC 主站
    "http://localhost:11451/*",       // 本地 API 服务器
    "file:///*"                       // 本地测试文件
  ],
  
  // 后台服务 Worker
  "background": {
    "service_worker": "scripts/background.js"
  },
  
  // 内容脚本配置
  "content_scripts": [
    {
      // 匹配规则：只在这些 URL 上运行
      "matches": [
        "https://member.szlcsc.com/member/order/*",
        "https://member.szlcsc.com/*order*",
        "file:///*test-page.html*"
      ],
      // 注入的 JS 和 CSS
      "js": ["scripts/content-script.js"],
      "css": ["styles/content-style.css"],
      // 加载时机：页面空闲时
      "run_at": "document_idle"
    }
  ]
}
```

**关键配置说明：**

| 配置项 | 作用 |
|--------|------|
| `manifest_version: 3` | 使用最新的 Chrome 扩展 API |
| `host_permissions` | 允许脚本访问的网站域名 |
| `content_scripts.matches` | 精确控制脚本在哪些页面执行 |
| `run_at: document_idle` | 等页面 DOM 加载完成后再执行脚本 |

### 2.3 background.js - 后台脚本（API 转发）

负责绕过 CORS 限制，将元件数据发送到本地 API 服务器。

```javascript
// 监听来自 content script 的消息
chrome.runtime.onMessage.addListener((message, sender, sendResponse) => {
  switch (message.type) {
    case 'addComponent':
      // 调用 API 转发函数
      handleAddComponent(message.data).then(result => {
        sendResponse(result);
      });
      return true; // 异步响应
  }
});

// 实际发送 POST 请求的函数
async function handleAddComponent(componentData) {
  const response = await fetch('http://localhost:11451/addComp', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(componentData)
  });
  return await response.json();
}
```

**为什么需要 background.js？**

由于 CORS 策略，网页无法直接请求不同域名的服务器。通过 background script 中转，可以绕过这个限制。

### 3.2 元素定位步骤

#### 第一步：找到订单列表容器

```javascript
// 选择器：.ant-spin-container > ul
const orderList = document.querySelector('.ant-spin-container > ul');
```

**原理：** LCSC 使用 Ant Design 的 spin 组件，订单列表在 `.ant-spin-container` 内部的 `<ul>` 元素中。

---

#### 第二步：找到每个订单项

```javascript
// 获取所有直接子元素 li
const orderItems = orderList.querySelectorAll(':scope > li');
```

**原理：** `:scope > li` 表示获取当前容器的直接子元素 `<li>`，避免选中嵌套的 li。

---

#### 第三步：解析订单基本信息

```javascript
// 订单号：通过链接元素
const orderNumEl = orderLi.querySelector('a[class*="0093E6"]');
const orderNumber = orderNumEl?.textContent.trim();

// 订单日期：第一个 section 的第一个 span
const headerSection = orderLi.querySelector('section:first-child');
const dateEl = headerSection?.querySelector('span:first-child');
const orderDate = dateEl?.textContent.trim();

// 收货人和总金额：信息区域的 li
const infoSection = orderLi.querySelector('[class*="18.9"]');
const receiverEl = infoSection?.querySelector('li:first-child span:last-child');
const totalEl = infoSection?.querySelector('li:last-child span:last-child');
```

#### 第四步：定位元件详情区域

```javascript
// 元件区域占 58% 宽度

```

#### 第五步：找到所有元件


#### 第六步：解析单个元件信息

每个元件下有两个 `<ul>`：
- 第一个：基本信息（名称、型号、制造商、编号）
- 第二个：价格信息（单价、数量、总价）

```javascript
// 获取所有 ul
const allUls = componentSection.querySelectorAll('ul');

// 第一个 ul：基本信息
const basicInfoUl = allUls[0];
const basicLiList = basicInfoUl.querySelectorAll('li');

// li[0]: 元件名称/类型
const fullName = basicLiList[0]?.textContent;
const componentType = fullName.split('/')[0].trim();  // "功率电感"
const componentName = fullName.split('/')[1].trim();   // "6.8uH ±20% 编带"

// li[1]: 型号/封装
const fullTypeText = basicLiList[1]?.textContent;
const model = fullTypeText.split('/')[0].trim();       // 型号
const packageType = fullTypeText.split('/')[1]?.trim(); // 封装

// li[2]: 制造商
const manufacturer = basicLiList[2]?.textContent.trim();

// li[3]: 元件编号 - 优先从图片链接的 title 获取
const imgLink = componentSection.querySelector('a img')?.closest('a');
const imgTitle = imgLink?.getAttribute('title');  // "6.8uH ±20% 编带(商品编号:C49308451)"
const partNumMatch = imgTitle.match(/商品编号[：:]([^)]+)/);
const partNumber = partNumMatch ? partNumMatch[1] : '';

// 第二个 ul：价格信息（通过文本内容定位，不依赖索引）
let priceUl = null;
for (const ul of allUls) {
  if (ul.textContent.includes('商品单价')) {
    priceUl = ul;
    break;
  }
}

// 从正确的 ul 中提取价格信息
const priceLiList = priceUl.querySelectorAll('li');
for (const li of priceLiList) {
  const text = li.textContent;
  if (text.includes('商品单价')) {
    priceText = text.replace('商品单价：', '').trim();
  } else if (text.includes('购买数量')) {
    quantityText = text.replace('购买数量：', '').trim();
  } else if (text.includes('商品金额')) {
    amountText = text.replace('商品金额：', '').trim();
  }
}

// 解析数值
const quantity = parseInt(quantityText.match(/(\d+)/)?.[1] || '0');
const unitPrice = parseFloat(priceText.match(/[\d.]+/)?.[0] || '0');
const totalPrice = parseFloat(amountText.match(/[\d.]+/)?.[0] || '0');
```

### 3.3 元素定位方法总结

| 方法 | 说明 | 示例 |
|------|------|------|
| `querySelector()` | 获取单个元素 | `document.querySelector('.ant-spin-container')` |
| `querySelectorAll()` | 获取多个元素 | `document.querySelectorAll('li')` |
| `textContent` | 获取元素文本 | `el.textContent` |
| `getAttribute()` | 获取属性 | `el.getAttribute('title')` |
| `closest()` | 向上查找父元素 | `img.closest('a')` |
| `:scope` | 限定选择器作用域 | `:scope > li` |
| `split('/')` | 分割字符串 | `"功率电感/6.8uH".split('/')` |
| `match()` | 正则匹配 | `"商品编号:C49308451".match(/商品编号[：:]([^)]+)/)` |
| `parseInt/parseFloat` | 转换为数字 | `parseInt("5个")` → 5 |

## 六、关键技术点

### 6.1 DOM 变化监听 (MutationObserver)

由于 LCSC 使用动态加载，订单展开后 DOM 会发生变化。使用 MutationObserver 配合 debounce 等待 DOM 稳定：

```javascript
function waitForDOMStable(callback) {
  return new Promise((resolve) => {
    let stableTimeout;
    let observer;
    
    const doResolve = () => {
      observer?.disconnect();
      clearTimeout(stableTimeout);
      callback();
      resolve();
    };
    
    const checkStability = () => {
      clearTimeout(stableTimeout);
      // 800ms 内没有新变化才认为稳定
      stableTimeout = setTimeout(doResolve, 800);
    };
    
    observer = new MutationObserver(checkStability);
    
    const container = document.querySelector('.ant-spin-container');
    if (container) {
      observer.observe(container, {
        childList: true,
        subtree: true,
        attributes: true
      });
    }
    
    // 5 秒超时保护
    setTimeout(doResolve, 5000);
  });
}
```

### 6.2 通过文本内容定位元素

避免依赖固定索引，改用文本内容匹配：


## 八、总结

本扩展的核心工作流程：

1. **页面加载** → manifest.json 匹配 URL → content-script.js 自动执行
2. **初始化** → 创建悬浮面板 → 绑定事件监听
3. **数据获取** → 展开订单 → 解析 DOM → 提取元件信息
4. **数据展示** → 渲染订单列表 → 渲染元件表格
5. **用户交互** → 点击添加 → background.js 转发 → API 存储

关键设计理念：
- 使用文本内容定位元素，避免依赖固定索引
- 使用 MutationObserver + debounce 处理动态 DOM
- 通过 background script 绕过 CORS 限制
- 清晰的模块化函数分工
